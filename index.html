## Особенности ## 1. Распределенность 2. Скорость 3. Гибкость
## Распределенность * Трудно угробить репозиторий, у кого-то останется свежая копия * Можно работать в оффлайне, вся история при вас
![Распределенность](distributed.png)
## Скорость Нет сетевых запросов, все операции (кроме push/fetch) локальны
## Гибкость Бранчи очень дешевы. Мерджи обычно легки. Позволяет выстраивать удобные вам деревья бранчей.
## Интерфейс Самый удобный интерфейс - командная строка. Даже если используете GUI, очень желательно понимать какие команды используются на самом деле.
## Отличия от SVN * Вся история на машине * Работаем с изменениями (changeset) а не версиями * Нет цифровых номеров ревизий, есть длинный идентификаторы * Нет единого update/commit. Есть fetch/push и pull/commit * Перед коммитом файл надо добавить в коммит (git add) * Авторизация чаще по SSH ключу, чем по login/password
## Командная строка Представляемся ``` $ git config --global user.name "Oleksandr Rysenko" $ git config --global user.email oleksandr@rysenko.com ``` Будет в описании коммитов, не используется для авторизации
Создаем репозиторий ``` $ mkdir repo $ cd repo $ git init ``` или ``` $ git clone git@github.com:rysenko/git-pre.git ``` и смотрим статус ``` $ git status ``` и смотрим историю ``` $ git log ```
Добавляем код ``` $ touch READ.md $ git add READ.md $ git commit -m "Initial" ``` Удаляем/переименовываем файлы ``` $ mv READ.md README.md $ git rm READ.md $ git add README.md ``` или ``` $ git mv READ.md README.md ```
Добавляем изменения (или меняем commit message) ``` $ git commit --amend -m "PRJ-100 Proper commit message" ``` Отменяем изменения в staged файле ``` $ git reset HEAD index.js ``` Отменяем изменения в не-staged файле ``` $ git checkout -- index.js ``` Откатываем коммит ``` $ git revert 123abc ```
Откатываемся на HEAD или чужую ветку (опасно) ``` $ git reset --hard HEAD $ git reset --hard upstream/branch ``` Удаляем файлы не под контролем версий ``` $ git clean -f ``` Игнорируем ``` $ echo ".idea" >> .gitignore ```
Удаленные репозитории ``` $ git remote -v origin git@github.com:rysenko/git-pre.git (fetch) origin git@github.com:rysenko/git-pre.git (push) ``` Пушим изменения ``` $ git push origin master ``` Ставим умолчательный remote ``` $ git push -u origin master ``` и потом просто ``` $ git push ``` Включить simple вместо matching (c 2.0 всегда) ``` $ git config --global push.default simple ``` Резолвим конфликты :) ``` $ git push -f ```
Забираем изменения со всех remotes ``` $ git fetch --all ``` Обновляем ветку (git fetch + git merge) ``` $ git pull $ git pull origin master ``` Обновляем с rebase (git fetch + git rebase) ``` $ git pull --rebase ```

Global State

Set data-state="something" on a slide and "something" will be added as a class to the document element when the slide is open. This lets you apply broader style changes, like switching the background.

Custom Events

Additionally custom events can be triggered on a per slide basis by binding to the data-state name. 


Reveal.addEventListener( 'customevent', function() {
    console.log( '"customevent" has fired' );
} );

Slide Backgrounds

Set data-background="#007777" on a slide to change the full page background to the given color. All CSS color formats are supported.

Image Backgrounds

<section data-background="image.png">

Repeated Image Backgrounds

<section data-background="image.png" data-background-repeat="repeat" data-background-size="100px">

Background Transitions

Pass reveal.js the backgroundTransition: 'slide' config argument to make backgrounds slide rather than fade.

Background Transition Override

You can override background transitions per slide by using data-background-transition="slide".
